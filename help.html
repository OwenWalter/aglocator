Help & Documentation

Everything you need to know about using AG Locator

Searching Entries

How to Search the AG Locator Database

To search for a product or keyword, use the Search page. When searching for entries, the database is searched based on the entry keyword field, assignment group name field, and notes field.

Search Tips

The AG Locator search is very literal. If you are searching for SAS Enterprise Guide, but the signature was submitted using a shorthand abbreviation such as "SAS EG", it may not appear in the search results.

Try various searches using different terms, abbreviations, or broader queries if you are unable to locate results you expect.

Remember also that the AG Locator database only includes the content provided by users. If you are unable to find an entry on a product or area you think would be useful, please make one and then submit it to the database for others to use!

Adding Entries

How to Add a New Entry to the AG Locator Database

To add a new entry, use the Add New Entry page. When adding a new entry, be sure to give it a good keyword/name that properly describes the relevant area. If you need to provide more information, the notes field can be used.

Supported Formatting and Input

The AG Locator database will preserve text spacing. It can preserve additional formatting if specified as valid HTML code.

Modifying Entries

How to Update an Entry

To update an entry, search for the entry in the database, then click its name to open the View Entry Details page for that entry. On the bottom-left of this page, find and click the link named "Modify Entry" to bring up the modification screen.

Make the desired edits to the signature text and then click the "Save Modified Entry" button at the bottom-left of the modification screen to update the signature in the database.

Note that this action will prompt you for your author key, a unique value based on the email address provided when the entry was originally submitted. Only the author key matching the email address provided with the original signature submission will be accepted, meaning that only the original entry author can update it.

If you do not know your author key, follow the prompts on the Modify Entry screen or click here to request your author key be emailed to you.

If the original author of an entry has retired, changed positions, does not have access to the saved email address, or otherwise is no longer available to modify it, please contact Andy Foreman for assistance.

To perform bulk updates on entries or bulk author updates, please contact Andy Foreman.

How to Remove an Entry

To remove an entry from the database, follow the same steps as seen in "How to Update an Entry" above, but instead click the "Delete Entry from Database" button at the bottom-right of the Modify Entry screen.

If you feel that an entry should be removed but you are not the original author of that entry, please contact Andy Foreman or the ARTS WebDev team.

The ARTS WebDev team reserves the right to remove entries at their discretion, as well as at the request of management. If possible, we will attempt to contact and allow authors to modify problematic entries before outright deleting them.

Entry Removal Policies

The development team reserves the right to edit or remove an entry from the database at any time, for any reason.

Entries containing inappropriate content, depreciated or inaccurate information, or other issues which have been deemed unacceptable may be modified or deleted without notification to the original submitter.

If you notice an entry that you feel should be removed due to its content, please report it by contacting the development team (provide a link to the problematic entry and explain why it should be removed).

Additional Help

What is AG Locator?

AG Locator is a database for SAS Technical Support and other SAS employees where associations between products, components, etc. and the relevant Assignment Groups (in ServiceNow CSM) be stored and accessed. We envisioned AG Locator as a way of enabling self-service management of these associations for Technical Support engineers and groups.

The name "cheatsheet" in the URL is a reference to a similar tool which was formerly part of SAS Technical Support's deprecated custom ticketing application, Sirius.
